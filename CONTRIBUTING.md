# 협업 규칙

1. `main` 브랜치에는 검증된 파일만 병합한다.
2. TCAD Golden Deck을 직접 복사해 여러 버전으로 만들지 않는다.
3. 파라미터는 가능한 한 공통 config 파일에서 읽도록 한다.
4. 수렴 실패를 삭제하지 말고 registry에 기록한다.
5. 좌표축은 x=Source→Drain, y=Fin 폭, z=Fin 높이로 고정한다.
6. 결과 그래프에는 단위, bias 조건, 온도, 구조 ID를 표시한다.
7. 문헌에서 가져온 구조나 수식은 출처를 남긴다.
8. 실제 Fab 검증이 아닌 결과는 `TCAD simulation` 또는 `Virtual proof-of-concept`로 표현한다.
